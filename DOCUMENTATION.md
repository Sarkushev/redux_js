# 🚗 Автомобильный портал Redux

## Описание проекта

Это полнофункциональное React + Redux приложение с:
- ✅ **JSON данные** - все данные загружаются из JSON файлов
- ✅ **Асинхронная загрузка** - с имитацией задержки (1-2 сек)
- ✅ **Несколько контентов** на главной странице
- ✅ **LIST/DETAIL функциональность** для статей
- ✅ **Redux Toolkit** для управления состоянием
- ✅ **Красивый UI** с плавными анимациями

## 🔧 API с имитацией задержки

Все функции в `src/api/api.js` имитируют сетевые запросы с задержкой:

```javascript
const simulateDelay = (ms = 1500) => {
  return new Promise(resolve => setTimeout(resolve, ms));
};

export const fetchCars = async () => {
  await simulateDelay(1500);
  return carsData;

```

Задержки:
- Автомобили: 1.5 сек
- Статьи: 2 сек
- Товары: 1.8 сек
- Категории: 1.3 сек

## 🎨 Redux управление состоянием

Каждый набор данных управляется своим Redux slice:

**userSlice.js** (Автомобили):
```javascript
{
  users: [],              // Список всех автомобилей
  currentUser: null,      // Выбранный автомобиль
  loading: false,         // Состояние загрузки
  error: null            // Ошибки
}
```

**articleSlice.js** (Статьи):
```javascript
{
  list: [],              // Все статьи
  selectedArticle: null, // Выбранная статья
  loading: false,
  error: null
}
```

Аналогично для productSlice и categorySlice.

## ⚡ Асинхронные операции

Все данные загружаются асинхронно с использованием `createAsyncThunk`:

```javascript
export const loadArticles = createAsyncThunk(
  'articles/loadArticles',
  async () => {
    return await fetchArticles();
  }
);
```

Это автоматически управляет состояниями:
- `pending` - показывает загрузку
- `fulfilled` - обновляет данные
- `rejected` - обрабатывает ошибки

## 🎯 LIST/DETAIL реализация (Статьи)

В компоненте `ArticleList.jsx`:
- Хранится `selectedId` в локальном состоянии
- Если `selectedId` не установлен - показывается список
- Если установлен - показываются детали выбранной статьи
- Кнопка "Вернуться к списку" сбрасывает выбор

## 📝 Примечания

- Все компоненты используют React Hooks (`useState`, `useEffect`)
- Redux управляет глобальным состоянием приложения
- Стили организованы в отдельных CSS файлах
- Используется CSS Grid для адаптивных макетов

## 🔄 Workflow загрузки

1. Компонент монтируется (`useEffect`)
2. Dispatch асинхронного действия (`loadCars`, `loadArticles`, и т.д.)
3. Redux переводит состояние в `loading: true`
4. API функция имитирует задержку с `setTimeout`
5. Данные возвращаются и сохраняются в Redux store
6. Компонент перерендеривается с новыми данными
7. Спиннер скрывается, отображаются данные
